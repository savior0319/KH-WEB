@ 인터넷이란?
-> 전 세계의 컴퓨터가 서로 연결되어 정보를 교환 할 수 있는 하나의 거대한 컴퓨터 통신망
-> 인터넷에서는 다양한 서비스를 제공함
      - Web Service, FTP Service, Mail Service 등등...

-> 프로토콜을 이용하여 통신규약을 정하고 통신을 함
      -Web = HTTP, Mail = SMTP, POP3, IMAP 등..

※ 웹(Web)이란?
-> 네트워크에 연결된 시스템의 "정보"를 한 위치에서 "검색" 할 수 있는 서비스를
-> Hyper Text를 통해 정보를 교환하는 서비스를
      - Hyper Text : 하나의 웹 문서에서 제공하는
      연결 주소(링크)를 통해 다른 문서로 이동 하는 것

@ Web Version
- Web 1.0 : 정적인 웹과 동적인 웹이 있음
  정적 웹 (초기 서비스 버전) - 고정적인 페이지를 제공
  동적 웹 (현재 가장 많이 사용되는 버전) - 동적인 검색 및 사용자의 참여, 공유 개방

- Web 2.0 : 지능형 웹 서비스 (사용자의 패턴을 분석하여 개인별 맞춤 정보 검색, 서비스 제공)
    - 사용자에게 필요한 지식만을 자동으로 검색하여 보여줌
    - facebook, youtube, instagram 등..

@ Web의 역사
-> 웹은 www라고 부르면 World Wide Web의 약자
-> 유럽의 입자물리학 연구소에서 얻어지는 엄청난 양의 연구 결과 및 자료의 효율적인 공유를 목적으로 1989년 웜 버너스리의 제안으로 개발 되었음
-> 1990년도에 웹이 등장하면서 인터넷 또한 급속도로 발전 하기 시작함
-> 웹을 이용하여 데이터를 주고 받기 위해 사용하는 언어는 HTML을 사용함

@ HTML (Hyper Text Markup Language)
-> Web Data(Web Page)를 전송하기 위해 사용되는 HyperText를 구성하는 Markup Language
  - Hyper Text : Hyper Link를 통해 구성된 형태로 다른 페이지로 쉽게 연결해주는 구조로 된 문서
  - Markup Language(마크업 언어) : 태그 등을 이용하여 문서난 데이터의 구조를 정의하는 언어

@ HTML의 역사
1. 인터넷의 시작
- ARPANET은 미국에서 소련에 인공위성 발사 성공에 대한 위협을 느껴 새로운 기술을 연구하였고, 이때 인터넷을 만들게 되었음(군사목적)
- 핵과 같은 공격에 대비하여 전체 통신 시스템에서 데이터를 안전하게 보관 및 전송 할 수 있는 시스템을 만들기 위함
-> 초기 ARPANET이 현재의 인터넷으로 발전 하게 됨

2. 웹의 등장
-> 유럽 공동 원자핵 연구소의 팀 버너스리의 제안으로 웹이 개발 되기 시작함
-> 추후 팀 버너스리 W3C 재단을 창설하게 됨
-> W3C는 웹 표준을 지정하는 기관(HTML 표준을 제정하는 단체)
-> W3C는 HTML 표준을 제정하지만 표준을 강제하지는 않음
-> 그로 인하여 웹 브라우저 전쟁이 일어나게 됨

3. 1차 웹 브라우저 전쟁
-> 기존 웹 서비스는 TUI 형태 였음
-> 이때, 모자이크라고 하는 최초의 GUI 모습을 갖춘 브라우저가 탄생 함
-> 모자이크는 점점 발전하면서 모자이크 커뮤니케이션에서 넷스케이프 커뮤니케이션으로 이름을 바꾸게 됨
    - 웹 브라우저 명칭도 모자이크에서 넷스케이프로 바꿈
-> 1994년 넷스케이프는 정식 버전을 발표하고 배포를 시작하게 됨
-> 이에 맞서 마이크로소프트가 인터넷 익스플로러를 발표함
-> 두 회사는 자신들의 웹 브라우저를 사용자가 쓰게 하기 위하여 W3C의 표준을 무시하고 자신들만의 기능을 추하게 서로의 웹 브라우저
점유율을 높이기 위해 노력 함
-> 인터넷 익스플로러가 아무리 노력해도 넷스케이프를 이기지 못하게 되자
마이크로소프트는 윈도우에 강제적으로 인터넷 익스플로러를 설치하여 배포 하게 됨
-> 추가적으로 마이크로소프트는 애플과의 계약을 통해 매킨토시의 기본 웹 브라우저도 인터넷 익스플로러로 설정함
-> 결국 넷스케이프사는 인터넷 익스플로러의 점유율을 따라잡기 힘들어 지고 1998년에 아메리카 온라인에 인수 됨

4. 플러그인
-> 웹 브라우저 전쟁으로 인해 웹은 엄청난 속도로 발전하기 시작함
-> 하지만 정작 웹 표준을 지정하는 W3C는 빠르게 대응하지 못함
-> 이에 불만을 느낌 기업들은 플러그인을 만듦
      - 플러그인은 웹 브라우저와 연동되는 특정 프로그램을 사용자 PC에 추가로 설치해 웹 브라우저 기능을 확장하는 방법
-> 1996년부터 현재 어도비 플래시, Active X 등은 웹을 점점 풍부한 기능을 가진 공간으로 만들어 줌
-> 그로 인하여 웹 브라우저를 통해 동영상 시청, 음악 감상 및 금융 업무등을 할 수 있게 됨


5. 웹 2.0 시대
-> 2000년대 부터 Active X를 기반으로 웹 애플리케이션들이 제작 되었음
-> 또한 플래시를 기반으로 일반 사용자를 위한 애니메이션들이 만들어지게 됨
    - 이때 졸라맨, 우비소년, 마시마로 등이 나오기 시작 함
-> 플래시와 Active X는 점점 발전 하게 됨
-> 인터넷은 점점 사용자끼리 뭉쳐 새로운 컨텐츠를 개발할 수 있는 공간으로 발전하게 되고
사용자끼리 정보 및 컨텐츠를 올려 창조 할  수 있게 된 웹 2.0 시대를 맞이 하게 됨


6. WHATWG의 등장
-> 전 세계 최고 점유율을 가진 인터넷 익스플로러는 사실상 W3C의 표준 웹 브라우저가 됨
-> 그러다 보니 모든 웹 사이트에 Active X 플러그인이 들어가게 되면서
웹 사이트는 무거워 지기 시작 함 (시작하기 전에 Active X 설치 해야 하므로)
-> 이를 막고자 2004년 W3C 회의에서 모질라재단과 오페라 소프트웨어는 새로운 HTML 표준 제안서를 W3C에 제출 함
      - 그러나 이 제안은 웹의 혁명을 위한 기존의 지형점에 위배 됨이라는 이유로 거절 당함
-> 그리하여 인터넷 익스플로러는 더욱 독점
-> 인터넷 익스플로러를 제외한 웹 브라우저 제공 기업들(애플, 모질라, 오페라)은 독자적으로 새로운 웹 표준 기관을 설립함(WHATWG의 등장)
-> WHATWG는 새로운 웹 표준으로 Web Application 1.0을 표준으로 작성하여 만듦(W3C를 무시)
-> 이와 비슷한 시기에 W3C는 마이크로 소프트사와 XHTML2.0 표준을 작성하였고 배포함

-> 실제 개발자들은 XHTML2.0을 사용하지 않고 WHATWG에서 만든 Web Application 1.0을 표준으로 선택함
-> 결국 W3C는 Web Application 1.0 표준을 HTML5 표준으로 변경하고 WHATWG는와 함께 HTML5를 발표함
-> HTML5 표준은 2012년에 초안을 발표하고 2014년 10월에 정식 권고안을 발표 하였음

7. 2차 웹 브라우저 전쟁
-> 결국 마이크로소프트와 W3C가 함께한 XHTML2.0표준이 붕괴되고 인터넷 익스플로러의 기능 문제가 대두 되었음
          - 다른 웹 브라우저는 모두 최신 표준을 지원하는데 인터넷 익스플로러는 오히려 최신을 지원하지 못하는 역현상이 발생함
-> 한마디로 지금까지의 웹 브라우저 점유율을 뒤집을 수 있는 기회가 생겨남
-> 이 시기를 놓치지 않으려고 모든 웹 브라우저가 빠른 속도로 웹 브라우저 업데이트를 진행
-> 실제 웹 브라우저 기업들은 개발 외에도 여러 마케팅 까지 하게 됨
          - 국내에는 인터넷 익스플로러가 점유율이 높아 다른 웹 브라우저 기업이 마케팅 하지 않으나 옆 나라 일본 같은 경우에는 유명한 캐릭터들을 앞세운 다양한 광고를 함
-> 심한 경우에는 해당 브라우저 아니면 사이트 접근 못하게 함
-> 현재도 2차 웹브라우저 전쟁 지속

@ HTML의 기본 구조

</doctype html> : 문서 유형
<html> : html의 시작
<head>
        문서의 각종 정보 문서 자체에 대한 설명 내용
        <title>, <meta>, <link>, <style>, <script> 등을 사용 함
</head>
<body>
        화면에 출력해서 보여주는 정보 및 내용
        head에 들어가는 태그를 제외하고 모든 태그를 사용
</body>
</html> : html의 끝


<!- -> : html의 주석

@ <head> </head>
-> 머리란 의미로써 html 코드를 시작하기 위해 필요한 내용들을  작성하는 공간

## <meta> tag ##
-> 헤더 내부에서 사용하는 태그로써 메타 데이터라는 뜻을 가지고 있음
-> html 문서가 가지고 있는 유용한 정보를 담아 두는곳
-> 문서 정보를 검색 엔진에 전달 함
-> 다양한 태그를 사용할 수 있음

문자셋 설정 : <meta charset="UTF-8">
웹 페이지에 대한 설명 : <meta name = "descript" content = "head 설명 페이지 입니다">
검색 엔진에 대한 키워드 : <meta name = "keywords" conent = "HTML, CSS, XML, JavaScript">
페이지 작성자 : <meta name = "author" content = "AHJ">
30초마다 문서 새로 고침 : <meta http-equiv = "refresh" content = '30'>

## <title> tag ##
-> 페이지의 제목을 나타내는 태그
-> 브라우저 창 이름을 설정 할 수 있음

작성법
<title> 타이틀 내용 </title>

## <script> tag ##
-> 페이지에서 스크립트를 사용하기 위해 쓰는 태그
      ※ 꼭 head에만 명시할 수 있는것은 아님(일반적으로는 head에 명시함)


작성법
<script> 스크립트 내용 </script>

## <Style> tag ##
-> 태그의 스타일을 지정하기 위해 사용하는 태그
-> CSS 속성들을 이용하여 적용하기 위해 사용함

작성법
<style> CSS 속성명 </style>

@ 글자 태그
-> 글자를 표현하기 위해 사용하는 태그

## h 태그 ##
<h?></h?>
-> 일반적으로 제목을 입력할 때 사용하는 태그
-> 글자의 크기를 조정 할 때 사용하는 태그
-> h뒤에 숫자로 구분함

작성법
<h?> 제목 내용 </h?>

1 > 2 > 3 > 4 > 5 > 6

## <br> 태그 ##
-> 문장의 줄 개행을 할 때 사용

## <hr> 태그 ##
-> 페이지에 가로 줄을 만들어 줄 때 사용

## <p> 태그 ##
-> 한개의 단락을 만들 때 사용
-> 자동으로 문장 개행 후 한 줄 공백을 만들어 줌

## <pre> 태그 ##
-> 입력한 그대로 출력할 때 사용
-> 띄어쓰기, 들여쓰기, 줄 비꿈이 입력된 그대로 출력 됨

## <strong> 태그 또는 <b> 태그 ##
-> 문장에서 문자를 강조하여 굵게 표시할 경우에 사용

## <em> 태그 또는 <i> 태그 ##
-> 문자에 기울임을 주는 태그

## <mark> 태그 ##
-> 배경 부분이 노란색이 되는 태그
-> 형광펜 표시가 된듯하게 출력

## <u> 태그 ##
-> 수평줄을 표시
-> <hr> 태그는 페이지 전체에 수평으로 줄을 표시하지만 <u> 태그는 해당 범위에만 수평줄을 표시

## <sub> 태그 ##
-> 아래 첨자 태그

## <sup> 태그 ##
-> 윗 첨자 태그

## <s> 태그 ##
-> 취소선 태그

## <cite> 태그 ##
-> 웹 문서나 포스트에서 참조 자료를 표시 할 때 사용하는 태그
-> 이탤릭으로 표시 (브라우저마다 상이함)
-> 기울임과 큰 차이는 없음 (다만 참조 사이트를 명시했는지를 찾아 볼 수 있음)

## <blockquote> 태그 ##
-> 다른 블로그나 사이트의 글을 인용할 경우 사용하는 태그
-> 자동 들여쓰기가 되어 다른 텍스트와 구별 할 수 있음

## <p> 태그 ##
-> blockquote 태그와 비슷하게 다른 블로그나 사이트의 글을 인용

## <ul> 태그 ##
-> 순서가 필요하지 않은 목록을 만들 때 사용
-> 리스트 앞에 특정 모양이 출력 됨 (기본 모양은 .)
-> 해당 모양은 CSS를 이용하여 수정할 수 있음
-> <li> 태그를 이용하여 목록을 표현함

작성법
<ui>
  <li>내용1<li>
  ...
</ui>

## <ol> 태그 ##
-> 순서가 있는 목록을 만들 때 사용
-> 속성으로 문자(abc../ABC), 숫자(123...), 로마자

속성
type : 순서형식을 정하는 것
start : 시작 순서

작성법
<pl type = '설정문자' start = '시작순서'>
  <li>내용1</li>
  ...
</pl>

## <di> 태그 ##
-> 용어나 문장에 대한 정의(definition) 리스트를 만듦
-> 설정 시 자동으로 들여쓰기가 됨

<dt></dt> : 정의의 제목
<dd></dd> : 정의 내용

<dl>
  <dt>정의제목</dt>
  <dd>정의내용</dd>
  <dd>정의내용</dd>
</dt>

@ 표 관련 태그
-> 표를 만들때 사용되는 태그

## table 태그 ##
-> 웹 문서에서 자료를 정리할때 자주 사용 (표 형태)
-> 행과 열로 이루어져 있고, 행과 열이 만나는 지점을 셀이라고 함

<tr> 태그와 <td> 태그를 이용함

<tr> : 한 개의 행(row)를 만드는 태그
<th> : 열에 대한 제목을 표시하는 태그, 중앙 정렬 및 굵게 표시 됨
<td> : 한 개의 열을 만드는 태그
